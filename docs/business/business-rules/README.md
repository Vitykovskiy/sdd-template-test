# Артефакты business-rules

## Назначение
Фиксирует правила, ограничения и decision logic для одного домена или bounded context.

## Единица декомпозиции
Один файл = один согласованный набор правил для одной предметной границы.

## Критерии разделения
- Разделяй по разному owner, policy set, lifecycle или области применения.
- Не смешивай в одном файле правила с разными источниками истины.
- Если правило меняет смысл в другом контексте, выноси его отдельно.

## Create vs update
- Если канонический файл уже есть и граница предмета не изменилась, делай `update`.
- Если появилась новая предметная граница, создай новый файл в `docs/business/business-rules/`.

## Минимальная готовность артефакта
- У каждого правила есть условие, действие или ограничение.
- Указана область применения.
- Есть исключения или явно зафиксировано, что их нет.
