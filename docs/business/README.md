# Навигатор бизнес-артефактов

`docs/business` — каноническое место для BA-артефактов. `README.md` семейств содержит подробные спецификации; этот файл нужен только для навигации.

## Семейства

| Семейство | Источник истины | Размещение |
|---|---|---|
| `vision` | Один канонический файл `vision` на одну `vision-boundary`. | `docs/business/vision/` |
| `glossary` | Один канонический файл глоссария на один bounded context или домен. | `docs/business/glossary/` |
| `business-rules` | Один канонический набор файлов на один набор правил или домен. | `docs/business/business-rules/` |
| `scenarios` | Один канонический файл сценария на один пользовательский путь или use case. | `docs/business/scenarios/` |

## Порядок навигации
Использовать порядок семейств `vision -> glossary -> business-rules -> scenarios`.

Перед редактированием любого артефакта прочитать соответствующий `README.md` семейства и следовать его правилам декомпозиции, размещения, именования и решениям create-vs-update.

## Именование и размещение
- Хранить долгоживущие бизнес-артефакты только в `docs/business/<family>/`.
- Использовать имена файлов в `kebab-case` по предметной области.
- Не создавать параллельные источники истины для одной и той же темы.
- Черновики и заметки, привязанные к задаче, не заменяют канонические файлы.
