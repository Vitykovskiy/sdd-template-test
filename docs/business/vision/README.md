# Артефакты vision

## Назначение
Фиксирует бизнес-цель, целевое состояние и границы изменения.

## Единица декомпозиции
Один файл = одна vision-сущность для одного продукта, инициативы или bounded context.

## Критерии разделения
- Разделяй, если меняется продукт, инициатива, аудитория или горизонт видения.
- Не объединяй независимые цели в один документ.

## Create vs update
- Если канонический файл уже есть, обновляй его.
- Если появилась новая vision-сущность, создай новый файл в `docs/business/vision/`.

## Минимальная готовность артефакта
- Есть формулировка цели и целевого состояния.
- Определены границы и область применимости.
- Зафиксированы ключевые предположения и критерий успеха.
