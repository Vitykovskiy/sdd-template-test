# Артефакты scenario

## Назначение
Фиксирует один бизнес-сценарий или пользовательский путь от триггера до результата.

## Единица декомпозиции
Один файл = один сценарий с одним trigger, actor, goal и outcome.

## Критерии разделения
- Разделяй по разному trigger, actor, goal или outcome.
- Если ветвление описывает самостоятельный сценарий, выноси его в отдельный файл.
- Не объединяй в одном документе независимые journeys.

## Create vs update
- Если сценарий уже описан и его граница не изменилась, делай `update`.
- Если появился новый сценарий или новая предметная граница, создай новый файл в `docs/business/scenarios/`.

## Минимальная готовность артефакта
- Есть trigger, actor, goal и outcome.
- Описан основной ход сценария.
- Зафиксированы альтернативы или исключения, если они есть.
