# create_github_repository.py

## Назначение

Скрипт создаёт GitHub-репозиторий для текущего локального Git-репозитория либо привязывает существующий GitHub-репозиторий к remote `origin`.

## Контекст применения

Используется на стадии `initialization`, когда локальный репозиторий уже существует, но GitHub repository ещё не создан или не привязан.

## Входные параметры

- `--owner`
  - владелец GitHub-репозитория;
  - по умолчанию берётся текущий аутентифицированный пользователь.
- `--name`
  - имя создаваемого репозитория;
  - по умолчанию совпадает с именем локальной директории.
- `--attach-existing-url`
  - URL существующего GitHub-репозитория, который нужно привязать вместо создания нового.
- `--visibility`
  - `private`, `public` или `internal`;
  - по умолчанию: `private`.
- `--remote`
  - имя remote;
  - по умолчанию: `origin`.
- `--push`
  - если указан, после создания выполняется push локальных ref.

## Внешние зависимости

- `git`
- `gh`
- общий модуль `.codex/routing/_shared/lib/common.py`

## Алгоритм работы

1. Находит корень репозитория.
2. Проверяет, что текущая директория уже является Git-репозиторием.
3. Если `origin` уже указывает на существующий GitHub-репозиторий, завершает работу без изменений.
4. Если передан `--attach-existing-url`:
   - валидирует URL;
   - убеждается, что целевой репозиторий существует;
   - создаёт `origin` или меняет его URL;
   - возвращает статус `attached`.
5. Если URL для привязки не передан:
   - определяет owner и имя нового репозитория;
   - проверяет, нет ли конфликта имени;
   - при конфликте возвращает `name_conflict` с вариантами разрешения;
   - при отсутствии конфликта вызывает `gh repo create`.
6. Возвращает итоговый JSON.

## Формат результата

Основные статусы:

- `already_exists`
- `attached`
- `created`
- `name_conflict`
- `error`

В ответе также могут быть:

- `repository_full_name`
- `repository_url`
- `remote`
- `conflict_type`
- `resolution_options`

## Коды завершения

- `0` — репозиторий уже был корректно привязан, успешно создан или успешно привязан существующий.
- `2` — пользовательский конфликт или некорректный вход.
- `3` — ошибка выполнения внешней команды.

## Важные инварианты

- Скрипт не инициализирует Git сам; он требует уже существующий `.git`.
- При конфликте имени не делает неоднозначный выбор автоматически.
- При привязке existing repository всегда проверяет его доступность через GitHub API/CLI.
